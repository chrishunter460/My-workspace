## Exhibit Base Defaults

Inherited by every exhibit type. Values here are the ultimate fallback — type-level
frontmatter overrides these, and block-level `formatting:` / `config:` overrides those.

### Block Structure

Every `de_funk` code block has four sections. Only `type` and `data:` are required.

```yaml
type: plotly.line          # required — catalog key (namespace.variant)

data:                      # what to fetch and how to shape it
  x: temporal.date
  y: securities.stocks.adjusted_close

formatting:                # how to present it (optional — uses type defaults)
  title: Daily Close
  height: 420

config:                    # how this block behaves in context (optional)
  id: price-chart
  config_ref: controls
  page_filters:
    ignore: [sector]
  filters:
    securities.stocks.ticker: [AAPL, MSFT]
```

### `formatting:` fields (common to all types)

| Field | Default | Description |
|-------|---------|-------------|
| `title` | — | Display title above the exhibit |
| `description` | — | Subtitle or caption below title |
| `height` | 400 | Chart/table height in pixels |
| `show_legend` | true | Show series legend |
| `interactive` | true | Enable hover, zoom, pan |
| `color_palette` | default | Visual color scheme for series |

`color_palette` values: `default` · `pastel` · `high_contrast` · `blues` · `reds`

### `config:` fields (common to all types)

| Field | Description |
|-------|-------------|
| `id` | Exhibit identifier — used by `config_ref` on other blocks |
| `config_ref` | Link to a `control.config` block by its `id` |
| `page_filters` | Object with optional `ignore: [list]` or `ignore: ["*"]` |
| `filters` | Exhibit-specific filter pass-throughs: `{domain.field: [values]}` |

To ignore **all** note filters on a specific exhibit:
```yaml
config:
  page_filters:
    ignore: ["*"]
```

### Active config footer

Every rendered exhibit shows a collapsible footer with currently active settings:

```
▶ Active: group_by=Ticker | date=2024-01-01→2024-12-31 | filter: ticker=AAPL,MSFT
```

Auto-generated by the plugin — no extra block config needed.
